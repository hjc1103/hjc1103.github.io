康城村的海妖怪谈

序

重重海浪下的波涛声，混杂着隐秘而鬼祟的咆哮声，不断地折磨着我不堪重负的耳膜。

仔细分辨，夹杂在那一阵声音中的，是它的脚步声。长着鳞片的脚掌一次又一次地拍打在地面上。随之而来的是它身上的那不知名的黏液滴落的声音…

“啪嗒”、“啪嗒”，那渎神之音的源头离我越来越近、越来越近……

我握紧了手中的枪，对准了脑袋——不是它的，而是我的脑袋。

我不敢再一次面对它，那长满了鳞片的人类身躯，勉强可以称作是头颅的东西上，满是死鱼才有的、那无神空洞的眼球。瞪着你，看到你内心最真实的罪恶。

混合着那声音传来的，还有那令人作呕的鱼腥味…

该死的鱼！我本以为我可以逃离这一切。现在看来，唯有扣动手上的扳机，才能结束我，这罪恶的一生了…

序章就到这儿了，其实连载这样的一部算不上小说的“随笔”一直是我心心念念想着的事。不一定会一直写下去，写到哪算哪儿吧

经过一天的研究，我又学到了怎样添加更加高级的看板娘！下面我来详细介绍一哈。

终极–互动看板娘

第一步

下载大神的文件并解压到本地博客目录的themes/next/source下，修改autoload.js文件。

将

const live2d_path = "https://cdn.jsdelivr.net/gh/stevenjoezhang/live2d-widget/";

改为

const live2d_path = "/live2d/";

autoload.js中的注释的绝对地址指的是，将资源打包放到hexo/theme/next/source中后，以hexo/theme/next/source为根目录（/）的绝对路径，一般我们下载下来的是master分支，那么修改路径就是/live2d-widget-master/xxx.js，autoload.js的最后一个函数initWidget;中的第二个参数不要变，是一个后台api。我选择将文件夹重命名为“live2d”，因此按照如上方式修改。

第二步

在/themes/next/layout/_layout.swing中,标签中新增如下内容，########为你自己的github账号：

<script src="https://########.github.io/live2d/autoload.js"></script>

标签中新增如下内容：一定一定要加上依赖！！！！！

<script src="https://cdn.jsdelivr.net/npm/jquery/dist/jquery.min.js"></script>
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/font-awesome/css/font-awesome.min.css"/>

第三步

在主题配置文件_config.yml中,新增如下内容：

live2d:
  enable: true

想修改看板娘大小、位置、格式、文本内容等，可查看并修改 waifu-tips.js 、 waifu-tips.json 和 waifu.css。

最终的效果在我的博客就能看到啦！

今天来试验一下看板娘功能！

看板娘

首先，用如下的命令添加一个live2d

输入以下的命令，下载相应的模型

模型名称选择请单击此处各个模型的预览请访问原作者的博客

打开站点目录下的_config.yml文件，输入以下代码

live2d:
enable: true
scriptFrom: local
model: 
    use: live2d-widget-model-haruto #模型选择
display: 
    position: right  #模型位置
    width: 150       #模型宽度
    height: 300      #模型高度
mobile: 
    show: true      #是否在手机端显示

设置好之后我们便拥有了一个卡通人物
